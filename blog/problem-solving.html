That time I was stuck on a simple problem

For question 3 of the Gradebook kata exercise, the task was to add a new property, testScores, to each student property in the gradebook. The value of testScores should be equal to the student's scores in the scores array.

I often write out (with pen and paper) the problem and the desired result before writing out the steps in pseudocode.

On pen and paper, I figured the problem could be solved by looping through the names in the students array and assigning their corresponding scores from the scores array.

Instead of using a for loop, I wanted to use an array method as I wanted more practise on these.

I googled the most suitable array method, which turned out to be forEach(). However after reading the MDN docs I couldn't get my head around how to access the index of the array (the i in the for loop).

🤷🏻‍♀️ For no particular reason, I thought the forEach() method could take only one parameter.

The kata challenges were great to practise Javascript and it provided many opportunities to get comfortable with being completely lost.

I learned to not get too frustrated and take a break away from the screen when going down a random google rabbit hole (eg, when looking up nested loops above). I relied heavily on pen and paper (my version of the rubber ducky method) to deconstruct problems, as well as psuedocode, and Googling.

That time I elegantly (or somewhat) solved a problem

It was super satisfying when I could solve a question:

  • a. within a few attempts
  • b. with not many lines of code

This didn't happen a lot 😅 but there was one instance of this in question 3.1 (Odds out) in Javascript Olympics where the question was to create a function that accepted an array of numbers then returned that array with only even numbers.

Granted, the question did hint to use an array method haha but it was one of the few times where I felt like I knew what I was doing, somewhat.

I was able to dissect the question with a couple of lines of pseudocode and it was cool to able answer this in one line using the .filter() array method and the remainder % operator.

Things I learned during this question:

  • 1. array methods can be amazing alternatives from loops
  • 2. It's super important to fully understand the question and the outcome it is asking for in order to solve it. For other challenges, I would try to scribble down possible steps before even understanding the question in its entirety.
