What are the differences between relative, absolute, and fixed positioning?

CSS technical Blog

19 January 2023

Positioning elements in CSS can be a bit of a mission!

So let's learn about the basics of the position property.

This post will cover:

  • 👉 What does the position property do?
  • 👉 position: relative;
  • 👉 position: absolute;
  • 👉 position: fixed;

What does the position property do?

  • 💡 The CSS position property defines the position of an element in a document.
  • 💡 The final position of the element is then determined using the top, right, bottom, and left properties.

FYI, before we get started: position: status; is the default position

  • 💡 By default, the browser shows elements one after another.
  • 💡 The order can be horizontal or vertical, depending on whether the elements are block or inline elements.
  • 💡 The top/right/bottom/left, and z-index properties do not apply.
  • - For example, block elements (like <div>s) start on a new line and are rendered vertically.
  • - Inline elements (like <span>) do not start on a new line, and are rendered horizontally.

👉 position: relative;

  • 💡 position: relative; is similar to static, but now we can move an element relative to its original position using the properties top/left/bottom/right.
  • 💡 This does not affect the position of any other elements.

  • Let's explain this in an example.
  • Consider the two <div>s in our HTML file. They are green and yellow (as per the CSS).
  • The yellow <div> has position set to relative, and has been placed 60px from the top (from its original position), and 100px from the left. The effect these movements are shown below:

👉 position: absolute;

  • 💡 elements set to position: absolute; are positioned relative to their closest positioned ancestor (if any, otherwise relative the <html> root element block).
  • 💡 the specified element is removed from the normal document flow, and other elements will behave as if that element is not in the document.
  • 💡 Once position: absolute; is set on an element, the properties top/left/bottom/right determines its ultimate position.

  • For example:
  • - We have three <div>s green, yellow and red.
  • - If we set the red <div> to position: absolute; the top and left properties will move the element relative to its closest parent element that has a set position.
  • - In the image below, the red <div> has position set to absolute, and has been positioned relative to the parent container: 120px from the top, and 100px from the left.

👉 position: fixed;

  • 💡 elements set to position: fixed; are also removed from the normal flow direction of a page (like position: absolute;)
  • 💡 However, unlike position: absolute; fixed elements are always positioned relative to the <html> element.

  • For example:
  • - Say we have a red and a green <div>
  • - and we set the green <div> to position: fixed; and place it 5px from the bottom and 5px from the right.
  • - The green <div> will be positioned from the bottom right of the document, and will stay fixed in the that spot even if we scroll.
